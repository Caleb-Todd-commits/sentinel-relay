# Final Report Contract

The final report is not just a generated summary. It is the audit artifact proving the multi-agent workflow.

## Required sections

- Executive summary
- Timeline
- Evidence
- Root cause
- Risk assessment
- Customer impact
- Actions taken
- Remediation
- Open questions
- Audit trail

## Source linkage

Every report section should reference at least one of:

- `evidenceIds`
- `sourceMessageIds`
- `auditTrailMessageIds`

This preserves traceability and lets judges see that the report was produced from agent collaboration, not from a hidden one-shot prompt.
